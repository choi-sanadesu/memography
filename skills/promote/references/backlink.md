# 백링크 전략 (보수적)

승격 후 언급 페이지에 엔티티 링크를 추가하는 규칙. "최소 침습"이 원칙.

---

## 기본 원칙

1. **`## 관련` 섹션에만 추가**. 다른 섹션이나 인라인 텍스트는 절대 수정하지 않는다.
2. **링크 1줄만 추가**. 설명·문장 생성 금지.
3. **중복 방지**. 이미 같은 경로 링크가 있으면 skip.
4. **경로 기반 링크 강제**. CLAUDE.md §8 규칙에 따라 `[[04. entities/{category}/{name}|{name}]]` 형식.
5. **수정 후 보고**. 어느 페이지를 어떻게 수정했는지 완료 리포트에 포함.

---

## 처리 순서 (페이지마다)

### 1. `## 관련` 섹션 찾기

정규식 패턴: `^##\s+관련\s*$` (대소문자·공백 flexible)

- 매치 있음: 해당 섹션의 본문 영역 파악
- 매치 없음: 섹션 생성 (아래 "섹션 생성" 참조)

### 2. 섹션이 있을 때

섹션 본문을 읽어 이미 같은 엔티티 링크가 있는지 확인.

#### 중복 체크 패턴

- 경로 전체 매치: `[[04. entities/{category}/{name}]]` 또는 `[[04. entities/{category}/{name}|...]]`
- basename 매치 (Obsidian 호환): `[[{name}]]` 또는 `[[{name}|...]]`

둘 중 하나라도 있으면 **skip** (이미 링크됨으로 판단).

#### 추가

섹션의 기존 리스트 항목 뒤에 새 줄로 추가:

```markdown
- [[04. entities/{category}/{name}|{name}]]
```

섹션이 비어 있으면 (섹션 헤더만 있음) 새 리스트로 시작.

### 3. 섹션이 없을 때

페이지에 `## 관련` 섹션이 없는 경우 생성.

#### 삽입 위치 결정

우선순위 순으로 배치 위치를 찾는다:

1. `## 참고 소스` 섹션이 있으면 **그 직전**에 삽입
2. 없으면 페이지 **맨 끝**에 추가

#### 삽입 형태

섹션 앞에 빈 줄 1개 보장:

```markdown
{이전 섹션 끝}

## 관련

- [[04. entities/{category}/{name}|{name}]]

{다음 섹션 시작 또는 파일 끝}
```

### 4. `updated:` 필드 갱신

백링크 추가한 페이지의 frontmatter `updated:` 필드를 오늘 날짜로 교체.

(엔티티 승격도 내용 변화이므로 갱신 대상)

## 엣지 케이스

### 페이지에 frontmatter가 없음

일어나면 안 되는 상태이지만, 방어적으로 처리:

- 백링크 추가는 진행
- `updated:` 갱신은 skip
- 리포트에 "frontmatter 없음 — /lint 권장" 힌트 기록

### `## 관련` 섹션이 여러 개

정상이라면 1개뿐. 여러 개면 첫 번째 섹션에만 추가. 리포트에 중복 섹션 경고.

### `## 관련 엔티티` 섹션이 있음 (§4.1 엔티티 템플릿)

`## 관련`과 `## 관련 엔티티`는 **다른 섹션**. 이 스킬은 `## 관련` 섹션만 다룬다.

- 소스 페이지에는 `## 관련` 섹션만 존재 (§4.2 템플릿)
- 엔티티 페이지는 `## 관련 엔티티`와 `## 참고 소스` 별도 존재 (§4.1)
- 백링크는 **소스·일반 페이지의 `## 관련`에만** 추가. 엔티티 페이지의 `## 관련 엔티티`는 이 스킬이 쓰지 않음 (Step 4에서 엔티티 생성 시에 이미 처리).

### 페이지 본문 중간에 이미 인라인 링크가 있음

본문 중에 `[[Smerz]]` 같은 평문 링크가 이미 있어도, `## 관련` 섹션 기준으로만 중복 판단.

- 본문 링크 있음, 관련 섹션 없음: 섹션 생성 + 링크 추가
- 본문 링크 없음, 관련 섹션에 있음: skip
- 둘 다 없음: 섹션 생성/활용 + 추가

이유: 본문 인라인 링크는 의도적 서술일 수 있고, 이 스킬이 파악하기엔 맥락이 부족. 관련 섹션은 **구조적 참조 리스트**이므로 안전하게 추가 가능.

### 위키링크 형식이 섞여 있음

볼트에 `[[Smerz]]`(basename)와 `[[04. entities/인물/Smerz|Smerz]]`(경로) 둘 다 사용되는 경우:

- **새로 추가하는 링크는 항상 경로 기반** (CLAUDE.md §8 규칙)
- 기존 basename 링크를 경로 기반으로 업그레이드하지 않음. 별도 마이그레이션 작업이 필요하면 사용자 요청 시 진행.

## 수정 보고 포맷

완료 리포트의 `백링크 추가` 항목:

```
백링크 추가: {N}개 페이지

- [[{path1}]] — 섹션 기존, 링크 추가
- [[{path2}]] — 섹션 신규 생성, 링크 추가
- [[{path3}]] — 이미 링크 있음, skip
```

skip도 리포트에 포함해 투명성 확보.

---

## 금지 사항

- 본문 중 평문 이름을 위키링크로 **치환 금지** (`Smerz` → `[[Smerz]]` 변환 X)
- `## 관련` 외의 섹션 수정 금지
- 새 설명문·요약 자동 생성 금지. 링크 1줄만.
- 페이지 재정렬·포맷팅 정돈 금지 (의도치 않은 diff 방지)

이 스킬은 **구조를 건드리지 않고 링크 1줄만 추가**하는 게 유일한 역할이다.
