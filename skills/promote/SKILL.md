# alexandria-keeper: promote

엔티티 승격. 볼트 전반에서 반복 언급되는 대상을 `04. entities/`의 단일 진실 소스로 올리고, 관련 페이지들의 `## 관련` 섹션에 일괄 백링크를 추가.

## 언제 쓰는가

- `/ingest` 완료 후 리포트에 "엔티티 후보"가 뜨면 그걸 처리할 때
- `/lint` 실행 후 Check 6(엔티티 승격 후보)에서 올라온 이름 처리
- 사용자가 특정 인물·브랜드·작품을 엔티티로 올리고 싶다고 직접 지정할 때
- 주기적 볼트 정비 — "요즘 반복 언급되는 이름들 엔티티로 올려줘"

## 입력

| 모드 | 예시 | 처리 |
|---|---|---|
| 이름 직접 지정 | `/promote Smerz` | 해당 이름만 처리 |
| 배치 리스트 | `/promote Smerz, Erika de Casier` | 순차 처리 (각각 확인 단계 거침) |
| 무인자 스캔 | `/promote` | 볼트 전체 스캔 → 후보 제시 → 사용자 선택 |

## 동작 원칙

- 대상 볼트의 `CLAUDE.md §4.1`(엔티티 템플릿)·`§8`(엔티티 관리 규칙)을 **읽어서** 기준으로 삼는다. 하드코딩 금지.
- **백링크는 보수적으로**: 언급 페이지의 `## 관련` 섹션에 링크 1줄만 추가. 인라인 텍스트 치환 금지, 본문 재작성 금지.
- **경로 기반 링크 강제**: `CLAUDE.md §8` 규칙에 따라 `[[04. entities/{category}/{name}|{display}]]` 형식.
- **분류 확신 없으면 확인**: category confidence가 low면 사용자 선택 제시.
- **중복 시 병합**: 이미 있는 엔티티는 새 참고 소스만 추가.

## 6단계 워크플로우

[references/workflow.md](references/workflow.md)에 상세 분기·에러 처리. 개요:

| # | 단계 | 처리 | B모드 확인 |
|---|---|---|---|
| 1 | 중복 체크 | `04. entities/**/{name}.md` 존재 여부 | — |
| 2 | 증거 수집 | 볼트 grep → 언급 페이지·컨텍스트 수집 | — |
| 3 | 카테고리 분류 | 5종 중 판단 | ✋ confidence low일 때만 |
| 4 | 엔티티 페이지 구성 | §4.1 템플릿 + 컨텍스트 합성 | ✋ **신규 생성 확인** |
| 5 | 백링크 업데이트 | 언급 페이지의 `## 관련` 섹션에 링크 추가 | 자동 (수정 요약 보고) |
| 6 | `index.md` + `log.md` 갱신 | 자동 | 자동 |

## 카테고리 (5종)

CLAUDE.md §8 기준:

- **인물** — 아티스트·크리에이터·업계인·디자이너 (솔로 아티스트 포함)
- **브랜드** — 엔터사·패션·라이프스타일·에이전시 (아이돌 그룹 포함)
- **작품** — 앨범·MV·전시·영화·책·디자인 프로젝트
- **개념** — 디자인 용어·기획 프레임워크·이론·방법론
- **플랫폼** — SNS·채널·툴·정보 출처

판단 규칙과 edge case는 [references/category.md](references/category.md).

## 백링크 전략 (보수적)

- **추가 위치**: 언급 페이지의 `## 관련` 섹션. 섹션이 없으면 생성.
- **절대 안 하는 것**: 인라인 텍스트 `Smerz`를 `[[04. entities/인물/Smerz\|Smerz]]`로 치환 금지. 본문 내용 수정 금지.
- **중복 방지**: 이미 같은 경로 링크가 있으면 skip.

상세 로직은 [references/backlink.md](references/backlink.md).

## 이름 정규화

- 영문 이름: 원어 표기 존중 (대소문자 유지). `Smerz`·`BTS`·`Erika de Casier` 그대로.
- 한글 이름: 공식 표기 우선. 변이(`뉴진스` vs `NewJeans`)가 섞여 있으면 공식 표기를 파일명으로 채택, 나머지는 별칭으로 frontmatter에 기록.
- 특수문자: Obsidian 파일명 금지 문자(`/`·`\`)만 회피. 나머지 Unicode OK.

## 완료 출력

```
✓ promote 완료

- 엔티티: {name} ({category})
- 신규 페이지: [[04. entities/{category}/{name}]]

- 백링크 추가: {N}개 페이지
  - [[{page1}]]
  - [[{page2}]]
  ...
- 병합된 소스 (있으면): 기존 엔티티 페이지에 참고 소스 N개 추가
```

## 한계 (v1)

- **동의어 병합 미지원**: `뉴진스`와 `NewJeans`가 각각 언급되면 사용자가 수동 정규화 후 호출해야 함. v2에서 alias 매핑 검토.
- **인라인 링크 업그레이드 미지원**: 본문 중 평문 이름을 위키링크로 바꾸는 건 이 스킬이 하지 않음. Obsidian의 "Existing only" 자동 제안에 의존하거나 수동 처리.
- **관계 그래프 부재**: 엔티티 간 관계(소속·협업 등)는 사용자가 `관련 엔티티` 섹션에 수동 기록. 자동 추론은 out of scope.
- **병합 모드의 제한**: 기존 엔티티 페이지가 있을 때 자동으로 맥락을 덧쓰지 않음. 새 `참고 소스` 추가 + `updated:` 갱신까지만.
