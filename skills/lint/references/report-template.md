# Lint 리포트 출력 템플릿

스킬이 최종 출력할 리포트의 고정 포맷. 스킬 구현이 이 틀대로 채워넣는다.

---

## 템플릿

```markdown
# 📚 Memography Lint 리포트

**날짜:** {YYYY-MM-DD}
**볼트:** {vault-root-path}
**검사 파일 수:** {N}

---

## 요약

| # | 체크 | 상태 | 이슈 수 |
|---|---|---|---|
| 1 | frontmatter 스키마 | {✅ \| ⚠️ \| ❌} | {N} |
| 2 | 깨진 위키링크 | {✅ \| ⚠️ \| ❌} | {N} |
| 3 | 고립 페이지 | {✅ \| ℹ️} | {N} |
| 4 | inbox 잔여 | {✅ \| ⚠️} | {N} |
| 5 | stale `updated:` | {✅ \| ⚠️} | {N} |
| 6 | 태그 카탈로그 위반 | {✅ \| ⚠️} | {N} |

**총평:** {한 줄 요약 — 예: "규칙 위반 2건, 우선 처리 권장" 또는 "이상 없음"}

---

## 1. frontmatter 스키마

{이슈 없으면 "이상 없음" 한 줄}

{fail 있으면:}

### ❌ Fail ({N}건)

- `{path}`: {구체 위반 내용}
  - 현재: `{값}`
  - 기대: `{범위 또는 형식}`
  - 제안: `{대체값}`

{warn 있으면:}

### ⚠️ Warn ({N}건)

- `{path}`: {구체 경고 내용}
  - 현재: `{값 또는 "(없음)"}`
  - 기대: `{권장 범위}`
  - 제안: `{대체값}`

---

## 2. 깨진 위키링크

{동일 패턴}

### ❌ {N}건

- `{source-path}` → `[[{target}]]`
  - 위치: 본문 | related.{bucket}
  - 상태: unresolved | ambiguous
  - 후보: {basename 동명 파일 경로 목록 또는 "없음"}

---

## 3. 고립 페이지 (참고)

본문 + `related:` 양쪽에서 백링크 0인 페이지. **반드시 문제는 아니므로** 검토만 권장.

- `{path}` (type: {type}, updated: {date})

---

## 4. inbox 잔여

{0이면 "✅ 비어 있음"}
{1+이면:}

**{N}개 잔여:**
- `inbox/{filename}` ({size}, 수정: {mtime}, {경과일}일 경과)

{3일+ 잔류 또는 10개 초과 시:}

⚠️ 우선 처리 항목:
- `inbox/{filename}` ({경과일}일 경과)
- 총 {N}개 — 임계 10개 {초과 | 이내}

→ `/classify`로 일괄 정리 권장.

---

## 5. stale `updated:` (기준: {N}일)

{동일 패턴}

- `{path}` — {N}일 경과
  - type: {type}
  - 제안: 내용 재검토 또는 updated 필드 갱신

---

## 6. 태그 카탈로그 위반

{이슈 없으면 "이상 없음" 한 줄}

{warn 있으면:}

### ⚠️ {N}건

- `{path}`
  - 미등록 태그: [`{tag1}`, `{tag2}`]
  - 현재 태그: [{전체 tags}]
  - 제안: `meta/tags.md` 추가 또는 기존 태그로 대체

### 신규 태그 후보 (빈도순)

- `{tag}`: {N}개 페이지에서 사용 → `meta/tags.md` 추가 권장
- `{tag}`: 1개 페이지 → 단발성, 기존 `{유사 태그}` 권장

---

## 제안 조치 (우선순위순)

{이슈 심각도·수정 비용 기준으로 정렬한 To-do. 자동 수정 금지 — 전부 사용자 확인 요함.}

1. **{조치}** (체크 #{N})
   - 대상: `{path}`
   - 이유: {한 줄}
   - 예상 작업: {1줄 수정 | 수동 판단 필요 | 링크 재작성 등}

2. ...

---

*이 리포트는 `memography:lint` 스킬이 생성. 파일을 수정하지 않음 (v0.1 기준).*
```

---

## 상태 아이콘 기준

- ✅ : 위반 없음
- ⚠️ : 위반 있음, 처리 권장
- ❌ : 심각한 위반, 우선 처리
- ℹ️ : 참고 정보 (위반은 아님)

**심각도 판정:**

| 체크 | ⚠️ 기준 | ❌ 기준 |
|---|---|---|
| frontmatter | summary 길이 위반, tags 권장 미만(<3) | 필수 필드 누락(type/tags/updated/summary), enum 위반(type·media), 파싱 에러, tags 상한 초과(>7), `type: source`인데 source URL 누락, type-위치 불일치 |
| 깨진 링크 | ambiguous | unresolved |
| 고립 페이지 | (항상 ℹ️, 경고 아님) | — |
| inbox | 1+ 잔여 + (3일+ stale 또는 >10개) | — |
| stale | 1+ 경과 | — |
| 태그 카탈로그 위반 | 미등록 태그 1+ | — |

---

## 구현 노트

- 템플릿은 markdown으로 바로 출력 가능. 중간 파일 저장 없이 스트림 출력.
- 리포트를 파일로 남기고 싶으면 사용자가 요청 시 별도로. 기본은 화면 출력만.
- 추후 volume이 커지면 JSON 구조화 출력 옵션 추가 검토 (C모드 자동 실행용).
