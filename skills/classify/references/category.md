# Classify 카테고리 분류 규칙

볼트 §1 폴더 구조와 §3 폴더별 역할을 기준으로 inbox 항목의 최종 위치 판단.

---

## Tier 1 — Top-level destination (5종)

볼트 §1 기준:

| 폴더 | 대상 |
|---|---|
| `entities/` | 인물·브랜드·미디어·작품·개념 — 단일 진실 소스 |
| `sources/` | 특정 소스(인터뷰·영상·기사) 단위 정리. `source:` URL 필수 |
| `cases/` | 사나의 프로젝트·작업 케이스 |
| `references/` | 큐레이션 자체가 가치인 레퍼런스 모음 (외부 링크 중심) |
| `notes/` | 방법론·가이드·인사이트 (출처 독립적) |

---

## Tier 1 판단 절차

### 1. 외부 소스 기반 (URL/이미지/외부 자료가 핵심)

inbox 항목의 frontmatter `source:` URL이 있거나 외부 자료가 본문 핵심이면:

| 조건 | destination |
|---|---|
| 인터뷰·영상·기사 등 단일 소스 정리 (가장 흔함) | `sources/` |
| 새 엔티티 자체를 정의 (예: 처음 등장하는 인물·브랜드 페이지) | `entities/{sub}/` (✋ 신규 생성) |
| 사나 본인 작업 케이스 (외부 자료지만 사나가 만든 것) | `cases/` |
| 여러 소스 큐레이션·레퍼런스 모음 | `references/` |

### 2. 개인 메모/인사이트 기반 (source URL 없거나 부수적)

| 조건 | destination |
|---|---|
| 방법론·가이드·일반 인사이트 (가장 흔함) | `notes/` |
| 사나 본인 케이스 회고 | `cases/` |
| 새 개념·이론 정의 | `entities/concepts/` (✋ 신규 생성) |

### 3. 이미지 (`media: image`)

| 조건 | destination |
|---|---|
| 무드보드·레퍼런스 컬렉션 | `references/` |
| 사나 작업 결과물·캠페인 비주얼 | `cases/` |
| 단일 이미지 + 비전 캡션·OCR 분석 | `sources/` (소스 단위) |
| 엔티티(인물·브랜드) 대표 이미지 | `entities/{sub}/`의 첨부 (페이지 자체는 별도 결정) |

## Tier 2 — entities/ 서브 (5종, 볼트 §1)

| 서브폴더 | 대상 |
|---|---|
| `people/` | 사람·그룹 (솔로·아이돌 그룹 모두) |
| `brands/` | 브랜드·레이블·에이전시 |
| `works/` | 작품 (앨범·MV·전시·영화·책·디자인 프로젝트) |
| `concepts/` | 개념·프레임워크·이론·방법론·업계 용어 |
| `media/` | 매거진·플랫폼·도구·채널 |

### 판단 기준 (서브폴더)

#### people/

- 솔로 아티스트 (Smerz, Erika de Casier, IU 등)
- 크리에이터·디렉터 (아트디렉터·프로듀서·포토그래퍼)
- 업계인 (기획자·A&R·기자 등 개인)
- 디자이너 개인
- **아이돌 그룹** (NewJeans·BTS·TWICE) — 볼트 §1: "사람·그룹" 명시. promote 시절의 "그룹=브랜드" 분류는 폐기

#### brands/

- 엔터사 (하이브·YG·SM·JYP·ADOR)
- 패션 브랜드 (발렌시아가·메종키츠네)
- 라이프스타일 브랜드 (이솝·MUJI)
- 에이전시 (매니지먼트·크리에이티브 에이전시·디자인 스튜디오)
- 레이블 (ADOR·PLEDIS — 모회사와 별도 엔티티)

#### works/

- 음악 (앨범·EP·싱글·MV)
- 전시·공연 (패션쇼·전시회·콘서트)
- 영상 (영화·드라마·다큐)
- 출판 (책·매거진 이슈)
- 디자인 프로젝트 (브랜드 리뉴얼·캠페인·포스터 시리즈 — **타인 작업 중 레퍼런스 가치 있는 것만 entities/works/. 사나 본인 작업은 `cases/`**)

#### concepts/

- 디자인 용어 (스위스 타이포그래피·맥시멀리즘·바우하우스)
- 기획 프레임워크 (A&R·송캠프·360 아이돌 기획)
- 이론·방법론 (디자인 사고·디자인 시스템·모션 디자인 원칙)
- 업계 용어 (K-POP 3.5세대·팬덤 경제·크리에이터 이코노미)

#### media/

볼트 §1: "매거진·플랫폼·도구". promote 시절 "플랫폼" 카테고리 + 매거진·도구를 통합:

- 매거진·채널 (피치포크·The FADER·W Korea·아이즈매거진)
- SNS 플랫폼 (인스타그램·트위터/X·틱톡·유튜브)
- 스트리밍 (스포티파이·멜론·넷플릭스)
- 작업 도구 (피그마·어도비·Notion)
- 커뮤니티 출처 (DC인사이드 갤·레딧 서브 — 반복 언급 시)

## Edge Cases

### 아이돌 그룹 vs 멤버

| 언급 패턴 | 폴더 |
|---|---|
| "뉴진스가 컴백", "BTS 월드투어" (그룹 전체) | `entities/people/` |
| "하니가 출연", "RM의 솔로" (개별 멤버) | `entities/people/` (별도 페이지) |

그룹과 솔로 멤버는 **별개 엔티티**, 둘 다 `people/`. 관계는 `related.entities` 상호 링크.

### 레이블·자회사

엔터사 내부 레이블(ADOR, PLEDIS 등)은 `brands/` (별도 엔티티). 모회사와 상호 링크.

### 콜라보·피처링 앨범

A·B가 공동으로 낸 앨범은 `works/` 1개. `related.entities`로 A·B 양쪽 링크.

### 솔로 활동하는 그룹 멤버

- 그룹 엔티티: `people/`
- 멤버 솔로 엔티티: `people/`
- 각각 독립 페이지, 상호 참조

### 디자인 스튜디오 vs 스튜디오 대표

- 스튜디오: `brands/` (에이전시)
- 대표 디자이너: `people/`
- 두 엔티티 분리, 상호 링크

### 인물이 자신의 이름으로 운영하는 브랜드

- "Rei Kawakubo" (개인) vs "Comme des Garçons" (브랜드)
- 두 엔티티 분리. 인물 엔티티의 `related.entities`에 브랜드 링크.

### 시리즈물

- 시리즈 전체 (예: "Get Up" 앨범)는 `works/` 1개
- 시리즈의 각 컴포넌트(수록곡)는 별도 엔티티 **안 만듦** (단발 언급에 그치는 한). 반복 언급될 때만 승격.
- 사나 작업 시리즈: `cases/{각 케이스}` + frontmatter `series:` 필드로 묶음 (예: `series: studio-sana`)

### 해외 아티스트 + 한국 표기 변이

- 파일명은 **원어 표기**: `entities/people/erika-de-casier.md` 또는 `Erika de Casier.md` (볼트 컨벤션 따름)
- 한글 표기(에리카 드 카지에르)는 frontmatter `aliases:` 리스트에 기록
- 볼트 내 검색이 한글로도 걸리게 지원

### 플랫폼 vs 브랜드 (Spotify·Instagram 등)

볼트 §1: 플랫폼·매거진·도구는 모두 `media/`로 통합. brands와 분리.

- **media/**: 정보·콘텐츠 출처 역할 (Spotify, Instagram, 피그마)
- **brands/**: 기업·조직 자체가 주제 (Spotify사의 큐레이션 전략 분석 → media/Spotify에 사실 누적, 브랜드 분석이 압도적이면 brands/Spotify로 분리)

같은 이름이 두 맥락에서 쓰이면 **중심 맥락으로 판정**. 나머지는 해당 엔티티 페이지의 본문에서 다룸.

### 이름이 같은 인물·브랜드

둘 다 분리. 파일명 disambiguation 또는 aliases 활용.

## 확신도 판정

### high (B모드 ✋ 불필요, 자동 진행)

- 폴더가 자명 (예: source URL 있음 + 외부 인물 인터뷰 → `sources/`)
- 단일 카테고리만 가능

### medium (자동 진행 + 보고)

- 두 폴더 가능, 한쪽이 뚜렷하게 우세 (예: `notes/` vs `references/` — 본문이 큐레이션 위주면 references)
- 60% 이상 확신

### low (사나 확인 필수, Step 2 ✋)

- 두세 폴더가 비등하게 가능
- Edge case에 해당
- 신규 엔티티 시사 → 자동 ✋ (신규 엔티티 생성 자체가 ✋ 트리거)

```
## 분류 판단 (#{N}, confidence: low)

inbox 파일: {file}
- summary: {summary}
- 정황: {tags}, source URL: {has/none}, media: {value}

**추천:** {top1} — {근거}
**대안:**
- {top2}: {근거}
- {top3}: {근거}

어느 폴더로 진행할까요? (또는 "보류"로 다음 호출 시 재시도)
```
