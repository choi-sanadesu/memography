# Query 출력 템플릿

`/query` 스킬은 세 가지 리포트 모드 중 하나로 답한다: **확정 / 부분 / 모름**.

모든 모드는 quote-heavy — 근거 파일의 실제 본문을 발췌해 답변을 뒷받침한다. 파일 경로는 Obsidian 위키링크 형식(`[[path]]`)으로 노출해 사용자가 바로 이동할 수 있게 한다.

---

## 1. 확정 답변 — 충분한 증거

Layer 1/2/3 중 하나 이상에서 confidence ≥ medium 이고, 질의 의도가 해소된 경우.

```
## 답변
{요약 2~5줄. 근거 파일이 실제로 말하는 것만 합성}

## 근거

### [[{path1}]] (frontmatter 매칭, confidence: high)
> {발췌 3~10줄. 원문 그대로}

### [[{path2}]] (백링크 추적, confidence: medium)
> {발췌}

### [[{path3}]] (본문 매칭, confidence: low)
> {발췌}

## 질의 해석
- 키워드: {extracted_keywords}
- 도메인 힌트: {domain_hints or "없음"}
- 매칭 레이어: {used_layers, e.g. "frontmatter, 백링크"}
```

### 규칙

- `## 답변` 은 근거 파일 발췌 범위 안에서만 합성. 발췌에 없는 내용 추가 금지.
- `## 근거` 는 최대한 원문을 그대로 인용. 길이 조절이 필요하면 `...` 로 생략을 표기.
- 근거 파일은 3~5개로 제한. 그 이상이면 가장 신뢰도 높은 것부터 선별.
- 레이어 라벨을 반드시 붙여 사용자가 증거 유형을 구분할 수 있게.

---

## 2. 부분 답변 — 증거가 있지만 불충분

Layer 탐색 결과 관련 파일은 나왔지만 confidence가 전부 low 이거나, 질의 일부만 해소된 경우.

```
## 부분 답변
{해소된 부분 요약}

## 근거

### [[{path1}]] (본문 매칭, confidence: low)
> {발췌}

## 주의
- {해소되지 않은 부분 명시}
- 근거의 신뢰도가 낮다: {이유, 예: "본문 단발 매치만 있음", "관계 연결이 간접적"}
- 질의를 더 구체화하면 정답률이 오른다: {제안}

## 질의 해석
- 키워드: {extracted_keywords}
- 매칭 레이어: {used_layers}
```

### 규칙

- `## 주의` 섹션을 반드시 포함. 사용자가 결과를 얼마나 믿을지 판단할 재료를 준다.
- 상충되는 근거가 있으면 양쪽 모두 인용하고 어느 쪽이 맞는지 단정 금지.

---

## 3. 모름 — 증거 0

Layer 1·2·3 모두 confidence = none.

```
## 모름

볼트에서 "{question}"에 대한 근거를 찾지 못했다.

- frontmatter 매칭: 0건
- 백링크 추적: 0건
- 본문 검색: 0건

### 질의 재구성 제안
1. {구체적 제안 1 — 예: 동의어·다른 표기}
2. {구체적 제안 2 — 예: 도메인 한정 제거 또는 추가}
3. {구체적 제안 3 — 예: 엔티티명을 더 넓은 범주로}

### 가능성
- 볼트에 아직 수집되지 않음 — `/ingest`로 소스 추가 후 재질의
- 엔티티는 있지만 승격 전 — `/promote`로 엔티티 페이지 생성 후 재질의
- 표기 불일치 — 한글·영문·별칭 중 다른 표기로 시도
```

### 규칙

- **추측 금지.** "아마 ~일 것이다" 같은 문장 금지. 볼트에 없으면 없다고만 말한다.
- 질의 재구성 제안은 가능한 한 구체적으로. 단순히 "다시 물어보세요"는 불충분.
- 다른 Alexandria 스킬(`/ingest`·`/promote`)로 해결 가능한 경우 안내.

---

## 공통 규칙

- 답변 언어는 질의 언어를 따른다 (한글 질의 → 한글 답변, 영문 질의 → 영문 답변).
- 파일 경로는 위키링크 `[[...]]` 형식만 사용. 절대 경로나 상대 경로는 사용자 가독성을 해친다.
- 발췌 인용은 `>` 블록쿼트로. 여러 줄이면 줄마다 `>` 붙이거나 한 블록으로.
- 읽기 전용 원칙 준수 — 이 출력은 메시지일 뿐, 어떤 파일도 생성·수정하지 않는다.
