# 이미지 입력 처리

이미지 파일(`.png`·`.jpg`·`.jpeg`·`.webp`·`.gif`)을 볼트 inbox로 편입할 때의 메타 추출·캡션·OCR 세부 절차. Step 1(로딩)과 Step 4(페이지 작성) 분기에서 참조.

---

## 입력 조건

### 지원 확장자

| 확장자 | 처리 |
|---|---|
| `.png` · `.jpg` · `.jpeg` · `.webp` | 정상 경로 |
| `.gif` | 첫 프레임만 분석 (다중 프레임·애니메이션은 v0.1 범위 밖) |
| `.heic` · `.heif` | `sips -s format jpeg` 선변환 후 처리. 변환 실패 시 사용자 보고 |
| `.svg` | 래스터 아님 — 텍스트로 `Read` (이미지 분기 아님) |
| 기타 (`.bmp`·`.tiff` 등) | `sips`가 지원하면 진행, 아니면 미지원 안내 |

### 크기 한계

- 10MB 초과 이미지는 `Read`가 거부할 수 있음. 사용자에게 리사이즈 요청.
- 해상도 자체는 제한 없음 (`sips`로 메타만 추출).

---

## 메타 추출

Step 1에서 이미지를 `Read`하기 **전**에 실행. 메타는 Step 4 frontmatter로 흘러간다.

### macOS 표준 경로

```bash
sips -g pixelWidth -g pixelHeight -g format {path}
stat -f '%SB' -t '%Y-%m-%d' {path}      # creation date
stat -f '%z' {path}                     # byte size (참고용)
```

`sips` 출력 파싱:

```
/path/to/file.png
  pixelWidth: 1920
  pixelHeight: 1080
  format: png
```

추출 결과:

| 필드 | 값 예 | frontmatter 키 |
|---|---|---|
| 파일명 | `screenshot.png` | (파일 경로로 대체) |
| 포맷 | `png` | `format` |
| 해상도 | `1920x1080` | `resolution` |
| 생성일 | `2026-04-21` | `captured` |

### `sips` 미보유 환경 폴백

```bash
file {path}           # 포맷 추정
stat {path}           # 생성일
```

`file` 출력에서 포맷과 해상도를 정규식으로 뽑을 수 있으면 뽑고, 아니면 **빈 값으로 진행**. 메타 추출 실패가 전체 ingest를 막지는 않는다.

### 볼트 밖 이미지

`sources.md`의 Step 3 규칙상 볼트 밖 이미지는 **복사 확인 전에는 손대지 않는다.** 메타 추출은 사용자가 복사에 동의한 뒤, 볼트 내 최종 경로에서 수행.

## 캡션 생성

### 도구

`Read` 툴로 이미지를 불러오면 Claude 비전이 내용을 파악한다. 이미지 자체를 모델 컨텍스트에 싣고 아래 기준으로 한글 캡션을 생성.

### 캡션 가이드

- **길이**: 1~2문장. 과도한 묘사 금지.
- **내용**: 주요 피사체 → 구도·배치 → 분위기·색감·맥락 순. 메타 수치(해상도·포맷)는 캡션에 넣지 않음 (frontmatter가 담당).
- **문체**: 대상 볼트의 기존 inbox/소스 페이지 톤에 맞춤 (대체로 담백한 명사구 중심).

### 금지 사항

- **추측성 인명 식별 금지.** 얼굴이 보여도 "○○로 보이는 인물"처럼 단정 금지. 사용자 브리프에 이름이 명시된 경우만 반영.
- **PII 노출 주의.** 스크린샷에 이메일·전화번호·카드번호·토큰 등이 찍혀 있으면 캡션·OCR 둘 다에서 해당 부분을 `[민감정보]`로 마스킹하고 사용자에게 보고.
- **저작권 주의.** 상업 콘텐츠의 정확한 복제·재배포 유발 묘사 금지 (예: 만화 페이지 전문 전사 등).

## OCR (선택)

### 자동 판단 기준

캡션을 생성하면서 Claude가 이미지에서 텍스트를 인지하면 다음 규칙으로 OCR 여부 결정:

| 인지한 텍스트 | OCR |
|---|---|
| 단어 3개 이상 또는 완결된 문장 | **자동 전사** |
| 로고·짧은 UI 라벨·메뉴 한 단어 | 스킵 (캡션에서 "로고가 보인다" 수준으로만 언급) |
| 판독 불가(저해상도·심한 왜곡) | 스킵 + 메모: "텍스트 있으나 판독 불가" |

### 전사 포맷

inbox 페이지의 `## 추출 텍스트 (OCR)` 섹션에 기록:

```markdown
## 추출 텍스트 (OCR)

> {원문 그대로}

- 언어: {ko | en | jp | ...}
- 비고: {필요 시 — 예: "손글씨, 일부 판독 어려움"}
```

- **원문 그대로**를 우선. 자동 번역하지 않음. 번역이 필요하다고 판단되면 한 줄 요약만 같이 제시하고 사용자에게 전체 번역 여부를 물어본다.
- 긴 문서 이미지(스캔본·슬라이드 등)는 전문 대신 **핵심만 요약 + 원본 raw 보관 경로 안내**. 전문 전사는 사용자 요청 시에만.

### PII 감지 시

앞선 "금지 사항"대로 마스킹 후 Step 4 확인 단계에서 별도 경고 표시:

```
⚠️ 이미지에 민감정보로 보이는 영역이 있어 마스킹했습니다: {라벨}.
포함 여부를 확인해 주세요.
```

## frontmatter & 본문 예시

```yaml
---
title: K-POP 비주얼 디렉션 무드보드
type: inbox
tags: [비주얼디렉션, 무드보드, K-POP]
updated: 2026-04-21
summary: 파스텔 톤 인물 클로즈업을 3×3 그리드로 배치한 K-POP 무드보드. 채도 낮춤 + 필름 질감으로 레트로 분위기 강조.
source:
media: image
format: png
resolution: 2048x1152
captured: 2026-04-19
---
```

> `type: inbox`는 임시 마커. classify Step 3에서 최종 type 확정.
> `media: image`는 Step 2 자동 추론 결과 (확장자 규칙).

본문 구성:

```markdown
![[raw/images/moodboard-2026-04.png]]

## 캡션

파스텔 톤의 인물 클로즈업 9컷을 3×3 그리드로 배치한 무드보드. 전체적으로 채도를 낮추고 필름 질감을 얹어 레트로한 분위기를 유지한다.

## 추출 텍스트 (OCR)

> "SPRING 2026 / MOOD BOARD / REF-017"

- 언어: en

## 띠니의 관찰

…
```

> `## 관련` 섹션은 선택. classify Step 4에서 구조화된 `related:` 필드(entities/sources/cases/references/notes 버킷)에 작성. 본문 `## 관련` 섹션은 사람 가독용 보조.

## 에러·한계

| 상황 | 처리 |
|---|---|
| `sips` 실행 실패 | 폴백(`file`·`stat`) → 그래도 실패하면 해당 메타 빈 값 |
| 파일 손상(`Read` 실패) | Step 1에서 중단, "이미지 파일 손상 가능" 보고 |
| 10MB 초과 | 사용자에게 리사이즈 또는 분할 요청 |
| HEIC/HEIF | `sips -s format jpeg --out {tmp.jpg}` 후 진행, 실패 시 보고 |
| GIF 다중 프레임 | 첫 프레임만 분석. 여러 프레임이 내용적으로 의미 있으면 본문에 명시 |
| EXIF GPS·카메라 기종 | v0.1 범위 밖 (요청 시 추후 검토) |
